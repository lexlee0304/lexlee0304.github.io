[c++]c++学习笔记:运算符重载

发表于 2018-02-28

c++中的运算符重载可以为运算符提供自定义的行为，即对于不同的类，相同的运算符可对应不同的操作。c++已经为一些类型提供了运算符重载，比如对于整型和浮点型数据以及字符串类的加法操作就表现出了不同的行为。用户也可以自定义运算符重载，从而为自定义类提供更加灵活方便的操作。

运算符重载的本质即将运算符的对应运算转变为对应的函数操作，当用户使用某一运算符时，就转至定义该运算符操作的函数。

1. 运算符重载的规则

  1. 在c++中，绝大多数运算符都可以实现运算符重载，只用少数运算符除外，这其中包括，类属关系运算符 “.” ,成员指针运算符“.*”, 作用域分辨符“::”和三目运算符“?:”。

  2. 重载后的运算符优先级和结合性都不会发生改变。

  3. 不能改变原运算符的操作对象个数，且重载的功能应与原有的功能类似。

  4. 不能创造新的运算符

  5. (),[],->及赋值运算符必须被声明为类成员

2. 运算符重载的形式

运算符可以被重载为成员函数，也可以以友元函数的形式重载为非成员函数。

2.1 运算符被重载为成员函数

二元运算符

对于两个对象 A 和 B 及某一二元运算符 op 进行 A op B 的运算，那么在类定义中如何将运算符重载为成员函数？

首先，运算符应作为对象A的类的成员函数，而对象B对应的类将作为该重载成员函数的形参，运算结果则作为函数的返回值。如对于 A op B 的运算，重载的成员函数应该是这样的：

Type operator op (Type name);

如对于+运算符, 形式为Type operator+(Type name);

一元运算符

一元运算符的形式相比二元元算符来说更加简单。

只需满足 

Type operator op ();

如对于！运算符，形式就可以为 Type operator!() const;这样的形式

2.2 运算符被重载为非成员函数

有些运算符必须被重载为非成员函数的形式，比如 << 和 >> 运算符，原因在与如果用成员函数重载 << 和 >> 运算符

比如这样

AType operator<<(Output &stream);

那么我们的调用形式将变成这样：

A << cout;

很明显，这样的调用形式同我们常规使用<<很不相同，所以我们应该避免使用成员函数重载。

那么该如何以非成员函数的形式重载操作符，通常应该以友元函数的形式进行重载。

比如对于 PhoneNumber 类，

class PhoneNumber
{
    friend std::ostream &operator<<(std::ostream &,PhoneNumber &);
private:
    std::string areaCode;
    std::string number;
}

友元函数的具体定义为

ostream &PhoneNumber::operator<<(std::ostream &output,PhoneNumber &num)
{
    output << "(" << num.areaCode << ")" << num.number << endl;
}

这样就可实现以非成员函数形式重载运算符。

[c++]c++学习笔记：多态（一）

发表于 2018-02-28

c++中的多态分为编译时的多态和运行时的多态。

多态行为需要通过绑定实现，所谓绑定，就是将定义的标识符同其对应的存储地址联系在一起的过程，而根据绑定所处阶段的不同，可以分为编译时的绑定和运行时的绑定，前者称为静态绑定，后者称为动态绑定。

1.运行时多态

C++中的运行时多态，即动态绑定行为主要通过虚函数实现。

在基类的类定义中通过在函数声明前添加virtual关键字即可将函数声明为虚函数成员，注意，虚函数声明只能出现在类定义的函数声明中，而不能在函数成员实现的时候。

在派生类中，可不用显式给出虚函数声明，编译器可以根据以下规则来判断派生类的一个函数成员是不是虚函数成员。

  1. 函数是否与基类有相同的名称

  2. 函数是否与基类的虚函数有相同的参数个数及相同的参数类型

  3. 函数是否与基类的虚函数有着相同的返回类型。

除了在基类和派生类中定义相应的虚函数成员，必须通过对象的指针或是引用调用相应的虚函数才能实现动态绑定，直接通过对象名访问虚函数则只能实现静态绑定，即根据对象名的类型调用相对应的虚函数。

通过在基类和派生类中分别定义相应的虚函数成员，程序在执行时即可动态的根据调用者的类型决定应该调用哪一个函数实现，即根据对象的类型动态决定对象的行为。

简单的demo

定义基类：

class Parent
{
public:
    virtual void function1();
};

void Parent::function1()
{
    cout << "Parent.function1()" << endl;
}

定义派生类：

class Child : public Parent
{
public:
    void function1();
};

void Child::function1()
{
    cout << "Child.function1()" << endl;
}

调用：

int main()
{
    Parent parent;
    Child child;

    Parent *parent_ptr = &parent;
    Parent *child_ptr = &child;

    Parent &parent_ref = parent;
    Parent &child_ref = child;

    //通过指针调用
    parent_ptr->function1();
    child_ptr->function1();

    //通过引用调用
    parent_ref.function1();
    child_ref.function1();

    return 0;
}

执行结果：

Parent.function1()
Child.function1()
Parent.function1()
Child.function1()

可以看到同时定义Parent类型的指针和引用，但由于指向的对象的类型不同，调用结果也不同，实现了运行时的多态。

纯虚函数及抽象类

抽象类可以为一群具有相似功能的类族提供通用的接口。

c++中可以通过在类定义中添加纯虚函数，使得该类成为抽象类，即抽象类是具有纯虚函数的类。

纯虚函数只要在虚函数声明之后添加 “ = 0 ”，即可完成纯虚函数的声明。

纯虚函数只需在类定义中提供纯虚函数声明，无需提供具体的函数实现。

又一个demo

类定义

class Shape
{
public:
    virtual void draw() = 0;
};

class Circle: public Shape
{
public:
    void draw();
};

void Circle::draw()
{
    cout << "Draw a circle." << endl;
}

class Rectangle: public Shape
{
    void draw();
};

void Rectangle::draw()
{
    cout << "Draw a rectangle." << endl;
}

执行：

int main()
{
    Shape *shapes[2];
    Rectangle rec;
    Circle cir;

    shapes[0] = &rec;
    shapes[1] = &cir;

    for(size_t i = 0; i < 2; i++)
        shapes[i]->draw();   

    return 0;
}

执行结果：

Draw a rectangle.
Draw a circle.
